# Contributing Guidelines

Thank you for your interest in contributing to AIToolbox! We welcome all contributions, from bug fixes to new features.

To ensure a smooth collaboration, please follow these guidelines:

## Getting Started

1. **Fork the repository** to your own GitHub account.
2. **Clone the repository** to your local machine.
3. **Create a new branch** for your feature or bug fix.
4. **Read the documentation**: Before making any changes, please read the documentation in the `/docs` folder, especially the `/docs/adsense` folder which contains critical information for AdSense compliance and content optimization.

## How to Contribute

### 1. Issues and Bug Reports

- Search existing issues before creating a new one.
- Provide a clear description and steps to reproduce any bugs.

### 2. Feature Requests

- Open an issue to discuss the feature before implementing it.
- Ensure the feature aligns with the project goals.

### 3. Pull Requests

- Keep your changes concise and focused.
- Ensure your code follows the project's style and conventions.
- Update documentation if necessary.
- Include a clear description of your changes in the PR.

## Development Rules

- **AdSense Compliance**: All new features and content must strictly follow the AdSense policies outlined in `docs/adsense`.
- **Code Quality**: Write clean, maintainable, and well-documented code.
- **Testing**: Test your changes thoroughly before submitting a PR.

## Questions?

If you have any questions, feel free to open an issue or reach out to the maintainers.
